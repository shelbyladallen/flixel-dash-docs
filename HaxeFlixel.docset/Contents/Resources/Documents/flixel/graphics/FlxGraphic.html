package flixel.graphics

class FlxGraphic

implements IFlxDestroyable

BitmapData wrapper which is used for rendering. It stores info about all frames, generated for specific BitmapData object.

Variables

read onlyassetsClass:Class<BitmapData>

Class name for the BitmapData

read onlyassetsKey:String

Asset name from openfl.Assets

read onlyatlasFrames:FlxAtlasFrames

Atlas frames for this graphic. You should fill it yourself with one of the AtlasFrames static methods (like texturePackerJSON(), texturePackerXML(), sparrow(), libGDX()).

bitmap:BitmapData

Cached BitmapData object

read onlycanBeDumped:Bool

Whether the BitmapData of this graphic object can be dumped for decreased memory usage, but may cause some issues (when you need direct access to pixels of this graphic. If the graphic is dumped then you should call undump() and have total access to pixels.

destroyOnNoUse:Bool

Whether we should destroy this FlxGraphic object when useCount become zero. Default is true.

read onlyheight:Int = 0

The height of cached BitmapData. Added for faster access/typing

read onlyimageFrame:FlxImageFrame

ImageFrame object for the whole bitmap

read onlyisDumped:Bool = false

Whether the BitmapData of this graphic object has been dumped or not.

read onlykey:String

Key in BitmapFrontEnd cache

persist:Bool = false

Whether this graphic object should stay in cache after state changes or not.

read onlytilesheet:Tilesheet

Tilesheet for this graphic object. It is used only for FlxG.renderTile mode

unique:Bool = false

Shows whether this object unique in cache or not.

Whether undumped BitmapData should be cloned or not. It is false by default, since significantly reduces memory consumption.

useCount:Int

Usage counter for this FlxGraphic object.

read onlywidth:Int = 0

The width of cached BitmapData. Added for faster access/typing

Methods

addFrameCollection(collection:FlxFramesCollection):Void

Stores specified FlxFrame collection in internal map (this helps reduce object creation).

Parameters:

collection

frame collection to store.

destroy():Void

Trying to free the memory as much as possible

dump():Void

Dumps bits of bitmapdata == less memory, but you can't read/write pixels on it anymore (but you can call onContext() (or undump()) method which will restore it again)

inline getEmptyFrame(size:FlxPoint):FlxFrame

Creates empty frame for this graphic with specified size. This method could be useful for tile frames, in case when you'll need empty tile.

Parameters:

size

dimensions of the frame to add.

Returns:

Empty frame with specified size which belongs to this FlxGraphic object.

inline getFramesCollections(type:FlxFrameCollectionType):Array<Dynamic>

Searches frames collections of specified type for this FlxGraphic object.

Parameters:

type

The type of frames collections to search for.

Returns:

Array of available frames collections of specified type for this object.

onAssetsReload():Void

Asset reload callback for this graphic object. It regenerated its tilesheet and resets frame bitmaps.

onContext():Void

Use this method to restore cached bitmapdata (if it's possible). It's called automatically when the RESIZE event occurs.

undump():Void

Undumps bits of bitmapdata - regenerates it and regenerate tilesheet data for this object

Static variables

staticdefaultPersist:Bool = false

The default value for the CachedGraphics persist variable at creation if none is specified in the constructor.

staticfromAssetKey(Source:String, Unique:Bool = false, ?Key:String, Cache:Bool = true):FlxGraphic

Creates and caches FlxGraphic object from openfl.Assets key string.

Parameters:

Source

openfl.Assets key string. For example: "assets/image.png".

Unique

Ensures that the bitmap data uses a new slot in the cache. If true, then BitmapData for this FlxGraphic will be cloned, which means extra memory.

Key

Force the cache to use a specific Key to index the bitmap.

Cache

Whether to use graphic caching or not. Default value is true, which means automatic caching.

Returns:

Cached FlxGraphic object we just created.
